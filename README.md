# BatchFFmpeg 批量视频处理工具

## 简介
BatchFFmpeg 是一个用于批量处理视频文件的实用脚本，支持两种模式：
- **merge 合并模式**：将文件名末尾带有序号（如 `-1`、`-2`）的视频文件合并为一个 MP4 文件
- **rename 重命名模式**：根据正则表达式提取文件名中的数字，重新编号后输出为 MP4 文件

## 功能
- 批量将 `.ts` 文件转换为 `.mp4`（调用 ffmpeg），`.mp4` 文件直接复制
- 合并序号连续的视频片段（序号须从 1 开始且连续）
- 按正则表达式提取数字重新排序并重命名文件
- 测试模式下仅打印操作预览，不做实际处理

## 依赖
- Python 3.6 及以上
- 需安装 [ffmpeg](https://ffmpeg.org/) 并配置到系统环境变量（仅处理 .ts 文件时需要）

## 使用方法

### 命令行参数
| 参数            | 说明                                             | 是否必需 |
|-----------------|--------------------------------------------------|----------|
| -i, --input     | 输入目录                                         | 必需     |
| -o, --output    | 输出目录                                         | 必需     |
| -m, --mode      | 处理模式：`merge`（合并）或 `rename`（重命名）   | 必需     |
| --pattern       | 重命名模式的正则表达式，须含一个捕获组（默认 `(\d+)`） | 否  |
| -t, --test      | 测试模式，仅打印操作预览，不做实际处理           | 否       |

### 示例

#### 合并模式
将 `video-1.ts`、`video-2.ts` 合并为 `video.mp4`：
```bash
python VideoProcessor.py -i "源文件夹路径" -o "目标文件夹路径" -m merge
```

#### 重命名模式
按文件名中的数字重新编号：
```bash
python VideoProcessor.py -i "源文件夹路径" -o "目标文件夹路径" -m rename --pattern "(\d+)"
```

#### 测试模式
仅打印操作预览，不做实际处理：
```bash
python VideoProcessor.py -i "源文件夹路径" -o "目标文件夹路径" -m merge -t
```

## 合并模式说明
- 文件名末尾须带有 `-序号` 格式（如 `video-1.mp4`、`video-2.mp4`）
- 同一组文件的序号须从 1 开始且连续，否则会报错
- 去掉末尾序号后文件名相同的文件会被合并为同一个输出文件

## 重命名模式说明
- `--pattern` 为正则表达式，须包含一个捕获组 `()` 用于提取数字
- 提取到的数字用于排序，输出文件按顺序重新编号（自动补零）
- 例：`(\d+)` 可匹配 `episode12.ts` 中的 `12`

## 文件结构
- `VideoProcessor.py`：主程序，包含所有处理逻辑与命令行入口

## 注意事项
- 输出目录若不存在会自动创建，但若已存在且非空则会报错
- 合并时会在输入目录下创建临时 `tmp` 文件夹，处理完后自动删除
